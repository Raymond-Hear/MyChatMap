## 2026-03-28 更新记录（v0.0.2）

### 版本策略调整

v0.0.2 收缩支持范围，**仅支持豆包和腾讯元宝**两个平台。移除 Kimi 和 DeepSeek 支持，移除诊断日志下载功能。

### 移除内容

| 移除项 | 原因 |
|--------|------|
| Kimi 适配器 (aic-kimi.js) | 暂不支持，后续版本考虑恢复 |
| DeepSeek 适配器 (aic-deepseek.js) | 懒加载/虚拟滚动导致历史消息抓取不全（6轮对话只抓到2轮），根因未解决 |
| 诊断日志下载功能 | 精简功能，降低复杂度 |
| Kimi/DeepSeek 域名匹配 | manifest.json 中移除对应 URL patterns |

### 变更文件清单

| 文件 | 变更 |
|------|------|
| manifest.json | 版本号 0.0.1→0.0.2；移除 kimi/deepseek 域名和适配器脚本 |
| content/adapters/aic-kimi.js | 删除 |
| content/adapters/aic-deepseek.js | 删除 |
| content/aic-main.js | 移除 Kimi/DeepSeek 适配器选择逻辑 |
| popup/popup.html | 移除诊断日志按钮；底部改为"适用于豆包和腾讯元宝"；版本号更新 |
| popup/popup.js | 移除 Kimi/DeepSeek 平台检测；移除 downloadDiagnosticLog 函数 |

### DeepSeek 问题备忘（供后续恢复参考）

DeepSeek 适配器存在历史消息抓取不全的问题（#35/#36/#37），根因是 DeepSeek 使用虚拟滚动/懒加载，页面打开时只有最近几轮消息 DOM 存在。可能的解决方向：
1. 模拟滚动触发加载全部历史消息 DOM
2. 利用 DeepSeek 内部 API 一次性获取完整对话
3. 用户滚动时逐步收集消息并合并到 IndexedDB
